\begin{abstract}
This report details the design, optimization, and evaluation of a Causal Transformer-based Reinforcement Learning agent in the MuJoCo \texttt{Hopper-v5} environment. We investigate the transition from standard Markovian assumptions (MDP) to Partially Observable environments (POMDP), identifying critical numerical and architectural failure modes---such as the sawtooth reward collapse, normalizer sensor crushing, poisoned student dialect mismatch in Reinforcement Learning from Human Feedback (RLHF), and sequence causal confusion in Algorithm Distillation (AD)---and introduce structural solutions (gradient clipping, Polyak update smoothing, Eternal Textbook replay, Jury ensembles, scheduled action masking, and xLSTM projection offloading). Our final evaluations demonstrate that causal memory models equipped with structured temporal clocks consistently outperform reactive networks, showing remarkable resilience to sensory failure, observation noise, and delayed feedback.
\end{abstract}

\section{Introduction}
Reinforcement Learning (RL) agents often fail in real-world scenarios due to the non-Markovian nature of environments (POMDPs). Standard architectures like Multi-Layer Perceptrons (MLPs) lack the memory required to reconstruct state variables from historical observations. 

This project explores the use of a Causal Transformer to solve these challenges. We bridge the gap between high-level architectural design and micro-level optimization, addressing constraints like GPU memory limits, CPU-to-GPU data copies, and numerical stabilization.

\section{Task 1: Transformer Architectures for Locomotion}

\subsection{The Foundation: Twin Delayed DDPG (TD3)}
Before implementing the Transformer, we established a robust Reinforcement Learning framework using \textbf{Twin Delayed DDPG (TD3)}. TD3 improves upon standard DDPG by addressing the overestimation bias of Q-values through:
\begin{enumerate}
    \item \textbf{Clipped Double-Q Learning:} Uses two independent critic networks, updating against the minimum predicted next-state value:
    \begin{equation}
    y = r + \gamma \min_{i=1,2} Q_{\text{target},i}(s', a')
    \end{equation}
    \item \textbf{Target Policy Smoothing:} Adds clipped noise to the target action to smooth the value landscape:
    \begin{align}
    a' &= \text{Clip}(\pi_{\text{target}}(s') + \epsilon, \text{action\_low}, \text{action\_high}) \\
    \epsilon &\sim \text{Clip}(\mathcal{N}(0, \sigma_{\text{target}}), -c, c)
    \end{align}
    \item \textbf{Delayed Updates:} The actor network and target weights are updated at a slower frequency than the critic networks (e.g., update ratio of 2:1) to allow the Q-value landscape to stabilize.
\end{enumerate}

\subsection{Engineering Optimizations and Stability Fixes}
To overcome these stability hurdles, we implemented a series of industrial-grade optimizations:

\paragraph{1. The "Sensor Crushing" Normalizer Bug:}
We discovered a critical defect in the online normalization module. The code was averaging 11 independent robotic sensors (representing root height, coordinate angles, joint velocities) across the features dimension. The math was performing:
\begin{equation}
\mu_{\text{crushed}} = \frac{1}{D}\sum_{d=1}^D \mu_d
\end{equation}
This squashed distinct states into a single scalar, blinding the agent and forcing the Transformer to guess the true physical vectors from sequence history. We refactored \texttt{model.py} using Welford's algorithm to update mean and variance vectors independently for each sensor dimension $d$:
\begin{equation}
\mu_{d,\text{new}} = \mu_{d,\text{old}} + \frac{n_b}{n_{\text{total}}} \left(\mu_{d,\text{batch}} - \mu_{d,\text{old}}\right)
\end{equation}

\paragraph{2. Gradient Clipping (Fuse 0.5):}
Noisy experience batches generated by an unstable hopping gait occasionally caused extreme gradients. We integrated gradient clipping into \texttt{td3.py}:
\begin{equation}
g_{\text{clipped}} = g \cdot \min\left(1, \frac{0.5}{\|g\|_2}\right)
\end{equation}
This acts as a mathematical fuse, preventing parameter spikes from corrupting the actor network. Both the Actor and Critic networks employ \texttt{clip\_grad\_norm\_} to prevent noisy batches from producing weight spikes.

\paragraph{3. Target Smoothing Decoupling ($\tau=0.0005$):}
To stabilize the target critics, we reduced the Polyak target update rate from $\tau=0.005$ to $\tau=0.0005$:
\begin{equation}
\theta_{\text{target}} \leftarrow \tau \theta + (1-\tau) \theta_{\text{target}}
\end{equation}
Slowing down target updates ensures the target networks remain stable, decoupling target estimation from noisy student policy fluctuations.

\paragraph{4. Causal History Buffer and Masking:}
To replace the standard MLP actor with a Causal Transformer, we updated the replay buffer to sample continuous sequences of past states and actions of length $L \in \{16, 32\}$ instead of isolated transitions. This provides the agent with the temporal memory needed to navigate. Inside the Transformer, a strict causal mask is applied to ensure the policy can only attend to past steps, preventing the agent from leaking future information during training.

\paragraph{5. Python-to-C++ Bottleneck (Pointer-Style Deque Hand-off):}
In the control loop, the agent maintains an online history using a Python \texttt{deque}. The original code cast the deque to a list before calling the model:
\begin{lstlisting}[caption=Old Inefficient Conversion]
action = policy.select_action(..., state_history=list(state_history))
\end{lstlisting}
This forced Python to perform a deep copy of the sequence references on the heap at every simulation step. We refactored the pipeline to pass the raw \texttt{deque} directly, deferring contiguous memory layout translation to NumPy's compiled C++ engine inside \texttt{select\_action}.

\paragraph{6. Physics-to-GPU Decoupling (2:1 Training Ratio):}
Backpropagation through a Transformer actor is computationally expensive. We decoupled physics simulation from training steps. The agent performs two simulation environment steps for every single gradient update (2:1 training ratio), cutting training time by $\sim$40\% with negligible impact on sample efficiency.

\paragraph{7. Evaluation Overhead Tuning:}
We identified that testing the agent for 5 full episodes every 5,000 steps consumed significant execution time. We scaled evaluation intervals to every 25,000 steps for 2 episodes, reducing evaluation overhead by 90\%. We also integrated auto-save checkpoints every 100,000 steps to protect long training runs.

\section{Task 2: The Robustness Frontier (POMDP)}

\subsection{2a \& 2b: Sensor Failure and Noise (The Blindfold Test)}
To evaluate the agent's resilience to sensory failures, we mask all velocity-related inputs (setting root velocities and joint velocities to zero). This forces the agent to reconstruct derivatives (velocities) from positional state histories, turning the Markov Decision Process (MDP) into a Partially Observable Markov Decision Process (POMDP). Additionally, Gaussian noise ($\sigma = 0.1$) is injected into the remaining observations to simulate realistic hardware sensor static. While reactive MLP-based policies collapse under these noisy, unobserved conditions, the causal context of the Transformer enables robust state estimation, calculating smooth joint movements without raw velocity inputs.

\subsection{2c: Credit Assignment Delay (The Lag Test)}
Real-world robots often suffer from reward delays (credit lag) due to slow physical measurements or sparse target signals. We evaluate the agent's temporal credit assignment capabilities by delaying the reward feedback by $K=10$ steps. The Causal Transformer's self-attention mechanism excels under this credit lag because it can directly associate past states and actions with delayed reward signals without relying on Markovian backpropagation through intermediate state transitions. By mapping long-term dependencies across the causal context window, the Transformer maintains stable locomotion under delayed feedback, whereas standard MLP policies degrade because their immediate Markovian reward correlations are broken.

\subsection{2d: Reinforcement Learning from Human Feedback (RLHF)}
Instead of relying on hardcoded environment rewards from the simulator, we align the Causal Transformer policy using human preferences. We train a \textbf{Jury Ensemble of three independent Reward Models} (Judges) on preference labels using the Bradley-Terry model. The probability that trajectory segment $\sigma_1$ is preferred to segment $\sigma_2$ is:
\begin{equation}
P(\sigma_1 \succ \sigma_2) = \text{Sigmoid}\left(\sum_t R_\psi(s_t^{\sigma_1}, a_t^{\sigma_1}) - \sum_t R_\psi(s_t^{\sigma_2}, a_t^{\sigma_2})\right)
\end{equation}
We optimize the Jury using Binary Cross-Entropy (BCE) Loss:
\begin{equation}
\mathcal{L}_{\text{BCE}}(\psi) = - \mathbb{E}_{(\sigma_1, \sigma_2, y)} \left[ y \log P(\sigma_1 \succ \sigma_2) + (1-y) \log P(\sigma_2 \succ \sigma_1) \right]
\end{equation}
where $y=1$ if segment $\sigma_1$ is preferred, and $y=0$ if $\sigma_2$ is preferred.

\paragraph{The "Poisoned Student" and Catastrophic Forgetting:}
Preference-based learning in continuous environments is highly unstable. In initial runs, the agent's performance collapsed to zero around step 50,000. We identified two core failure modes:
\begin{itemize}
    \item \textbf{The Poisoned Student:} At the start of training, the agent's normalizer is uncalibrated, reporting sensors in an unfamiliar range (dialect). The Judge fails to recognize this dialect and grades randomly, and updating the Judge on these noisy rollouts corrupts its representations.
    \item \textbf{Catastrophic Forgetting:} Because the Judge is updated online on the student's latest noisy trajectories, it quickly forgets the characteristics of expert movement, leading to reward hacking.
\end{itemize}

\paragraph{Architectural Alignment and Hardening:}
To stabilize the training run, we implemented five key optimizations in \texttt{train\_rlhf.py} and \texttt{reward\_model.py}:
\begin{enumerate}
    \item \textbf{Sensory Normalization Synchronization:} We forced the student network to inherit the pre-trained expert $L=32$ normalizer parameters at initialization, aligning their sensory ranges.
    \item \textbf{The Jury Ensemble (Pessimistic Consensus):} We query three independent reward models in parallel and take the conservative minimum:
    \begin{equation}
    R_{\text{Jury}}(s,a) = \min\left(R_{\psi_1}(s,a), R_{\psi_2}(s,a), R_{\psi_3}(s,a)\right)
    \end{equation}
    Taking the minimum reward dilutes outlier predictions, preventing the policy from exploiting reward-model loopholes.
    \item \textbf{Reward Governor (Scaling and Clipping):} To prevent gradient explosions, we scale the Jury's predictions by $50.0$ and clip them to $[-100, 100]$ before feeding them to the policy.
    \item \textbf{Paced updates:} We update the Jury's weights only once every 1,000 steps, giving the policy time to adapt to reward updates.
    \item \textbf{The Eternal Textbook Protocol (Continuous Replay):} During online updates, we force the Jury to train on a mixture of expert pre-training trajectories (\texttt{old\_buffer}) and the student's live trajectories (\texttt{new\_buffer}). This continuous study ensures the Jury never forgets expert hopping dynamics.
\end{enumerate}

\paragraph{Analysis of the Late-Stage Performance Crash:}
As shown in Figure \ref{fig:rlhf_extended}, while our alignment strategies successfully delay the policy collapse beyond the initial $50,000$ steps (compared to the rapid sawtooth collapse of the unoptimized setup in Figure \ref{fig:rlhf_sawtooth}), the agent still collapses toward the end of the extended $300,000$-step run. This late-stage failure is driven by three primary factors:
\begin{itemize}
    \item \textbf{OOD Reward Hacking:} The policy eventually drifts into out-of-distribution (OOD) state-action regions and exploits false-positive high predictions from the Jury, adopting physically unstable gaits that lead to immediate balance collapse.
    \item \textbf{Normalizer Statistics Drift:} Continuous exploration shifts the running state normalizer statistics away from the distribution the Jury studied during expert pre-training. This sensory shift ("dialect mismatch") corrupts the Jury's reward predictions.
    \item \textbf{Jury Representation Drift:} Continuous fine-tuning on highly correlated student trajectories causes the Jury to overfit to recent rollouts, slowly forgetting the boundaries of expert hopping.
\end{itemize}

\subsection{2e: Attention Attribution (Chefer Protocol)}
To verify attention behavior, we implemented the \textbf{Chefer Protocol} (Transformer Relevancy Propagation) in \texttt{model.py}. Standard attention maps can fail to capture information flow because they only record forward activations. The Chefer formulation propagates gradients and attentions to compute a relevancy score $R$:
\begin{equation}
R^{(l)} = R^{(l-1)} + \bar{A}^{(l)} \odot \nabla A^{(l)}
\end{equation}
We resolved forward-only limitations by replacing the standard positive-gradient clamp with an \textbf{Absolute Saliency Engine}:
\begin{equation}
S_{i,j} = \left| A_{i,j} \odot \frac{\partial \mathcal{L}}{\partial A_{i,j}} \right|
\end{equation}
By incorporating absolute values, we captured inhibitory attention features---moments in history that signal the actor to halt or decelerate.

\subsection{Attention Saliency and Entropy Analysis}
We implemented a \textbf{Saliency Attribution Engine} and found that the agent exhibits "Action Shots"---sharp focus on the moments exactly 0.5s prior.

\paragraph{Attention Saliency Inversion: Clean vs. Blindfolded Models:}
By analyzing attention maps, we observed that when a model trained on clean observations is blindfolded (velocity sensors removed), its attention pattern inverts. Instead of utilizing historical data to estimate speed, it exhibits a "Panic Response"---focusing exclusively on the present timestep and ignoring the past.

To address this distribution shift, we conducted a \textbf{Robustness Sprint}---20,000 steps of training under masked-velocity conditions. Following this sprint, the model developed a double-anchor attention strategy:
\begin{enumerate}
    \item \textbf{The Calculus Spike (Step -1):} The model focuses intensely on the frame immediately preceding the current state to estimate velocity:
    \begin{equation}
    v_t \approx \frac{x_t - x_{t-1}}{\Delta t}
    \end{equation}
    \item \textbf{The Deep Memory Anchor (Step -31):} The model allocates $\approx$25\% more attention to the oldest step in its context window. This anchor serves to prevent drift in joint angle tracking.
    \item \textbf{Middle-Buffer Noise Filtering:} The model filters out the middle indices of the history window, focusing only on the boundary parameters.
\end{enumerate}

\paragraph{Attention Entropy Tracking:}
We track and plot the attention entropy $H(A_t) = -\sum_j A_{t,j} \log A_{t,j}$ over the course of training. During the initial stages of training, the attention entropy is high ($\approx \log(32) \approx 3.46$), indicating that the attention weights are uniform and the model is searching the entire context history. As the policy converges and develops the double-anchor strategy, the entropy drops significantly, stabilizing around $\approx 1.8 - 2.2$. This reduction in entropy indicates that the attention has focused onto specific critical historical frames (specifically the Calculus Spike at $t-1$ and the Deep Memory Anchor at $t-31$), filtering out sequence redundancy.

\section{Bonus Tasks}
This section details the design and evaluation of the bonus tasks from the assignment handout, focusing on positional encoding ablations, combined stressor challenges, in-context learning, and recurrent sequence models.

\subsection{Bonus A: Positional Encoding Ablation}
We benchmarked three variants of Positional Encodings (PE) on the hidden-velocity setting to understand how temporal ordering affects Causal Transformer policies:
\begin{itemize}
    \item \textbf{Learned Positional Encodings}: The model learns a lookup table of parameter embeddings for each index $0, \dots, L-1$ during training. While simple and adaptive, learned encodings are constrained by the maximum context length seen during training and cannot generalize or extrapolate to longer sequence windows at test time.
    \item \textbf{Sinusoidal Positional Encodings}: Fixed, deterministic sine and cosine functions of varying frequencies representing a temporal clock:
    \begin{align}
    PE_{(pos, 2i)} &= \sin\left(\frac{pos}{10000^{2i/d}}\right) \\
    PE_{(pos, 2i+1)} &= \cos\left(\frac{pos}{10000^{2i/d}}\right)
    \end{align}
    where $pos$ is the history index, $i$ is the dimension index, and $d$ is the embedding dimension. This provides a smooth, continuous geometric structure that generalizes well and acts as a stable "Temporal Ruler."
    \item \textbf{Rotary Positional Embeddings (RoPE)}: Rather than adding positional vectors directly to the inputs, RoPE applies a rotation matrix to the Query and Key vectors in the complex plane at each self-attention layer. This naturally encodes the relative distance between tokens geometrically and preserves translation invariance, enabling robust long-horizon dependency tracking.
\end{itemize}
Our evaluations, shown in Figure \ref{fig:pos_enc_ablation}, demonstrate that without positional encodings, the Transformer remains "time-blind" and fails to calculate the temporal derivatives needed for balance. Sinusoidal Encodings provided the most stable performance, preventing policy collapse and allowing the agent to settle into a sub-millimeter balance.

\subsection{Bonus C: In-Context RL via Algorithm Distillation}
In sequence-conditioned models like Algorithm Distillation (AD), we predict action $a_t$ based on historical state-action-reward tokens.

\subsubsection{Scheduled Action Masking (Curriculum Action Dropout)}
In continuous control environments, sequential actions are highly correlated ($a_t \approx a_{t-1}$). Consequently, the Transformer can develop a causal shortcut---ignoring state inputs and simply copycatting the previous action $a_{t-1}$. During online evaluation, this shortcut causes errors to compound rapidly, causing the agent to fall.

To prevent this, we implemented \textbf{Scheduled Action Masking}. The action dropout probability $p_{\text{mask}}$ increases linearly across epochs:
\begin{equation}
p_{\text{mask}} = \min(0.2 + 0.1 \times \text{epoch}, 0.8)
\end{equation}
This curriculum forces the network to transition from copying easy sequence patterns in early epochs to predicting actions solely from state histories in later epochs.

\subsubsection{History Jitter Data Augmentation}
We inject Gaussian noise into the historical state, action, and reward context buffers during training:
\begin{align}
\tilde{s}_\tau &= s_\tau + \eta_s, \quad \eta_s \sim \mathcal{N}(0, 0.01) \\
\tilde{a}_\tau &= a_\tau \cdot M_\tau + \eta_a, \quad \eta_a \sim \mathcal{N}(0, 0.005) \\
\tilde{r}_\tau &= r_\tau + \eta_r, \quad \eta_r \sim \mathcal{N}(0, 0.01)
\end{align}
where $M_\tau$ is the active scheduled action mask. Crucially, noise is applied \textbf{only to the input contexts}; the target loss is computed against clean, unmasked actions, forcing the model to reconstruct optimal actions from noisy histories.

\begin{lstlisting}[caption=Scheduled Masking and History Jitter Implementation]
# Calculate scheduled masking probability
p_mask = min(0.2 + 0.1 * epoch, 0.8)
mask = (torch.rand(actions.shape[:-1], device=device) > p_mask).unsqueeze(-1).float()
masked_actions = actions * mask

# Add noise to input contexts
jittered_states = states + torch.randn_like(states) * 0.01
jittered_actions = masked_actions + (torch.randn_like(actions) * 0.005) * mask
jittered_rewards = rewards + torch.randn_like(rewards) * 0.01

# Compute loss against clean targets
with autocast():
    pred_actions = model(jittered_states, jittered_actions, jittered_rewards, timesteps)
    loss = criterion(pred_actions, actions)
\end{lstlisting}

\subsubsection{Sequence AMP and Gradient Accumulation}
Attention scales quadratically with sequence length $\mathcal{O}((3L)^2)$, making sequence batches memory-intensive. 
\begin{itemize}
    \item \textbf{Automatic Mixed Precision (AMP):} Casts attention matrix multiplications to half-precision (\texttt{float16}) using \texttt{autocast} and \texttt{GradScaler} to prevent floating-point underflow.
    \item \textbf{Gradient Accumulation:} Splits a large batch size of 512 into 4 sequential micro-batches of size 128, accumulating gradients before running an optimizer step:
    \begin{equation}
    g_{\text{accumulated}} = \frac{1}{N_{\text{steps}}} \sum_{i=1}^{N_{\text{steps}}} \nabla_{\theta} \mathcal{L}_i
    \end{equation}
    This enables training with large effective batch sizes within consumer VRAM limits.
\end{itemize}

\subsection{Bonus D: xLSTM as Policy Backbone}

\subsubsection{xLSTM Recurrent Loop Projection Offloading}
In our custom xLSTM implementation (\texttt{xlstm\_model.py}), the sLSTM and mLSTM recurrent cells loop sequentially over time:
\begin{equation}
h_t, C_t, n_t, m_t = \text{Cell}(x_t, h_{t-1}, C_{t-1}, n_{t-1}, m_{t-1})
\end{equation}
Computing linear projections ($Q, K, V$ and gating weights) inside this loop incurs significant GPU kernel launch overhead. We optimized this by computing the projections in parallel \textbf{outside the recurrent loop} and slicing the resulting tensors inside the loop:

\begin{lstlisting}[caption=xLSTM Projection Offloading Optimization]
# Pre-compute projections outside loop
Q_all = self.q_proj(X_conv)      # Shape: (B, L, H * d_k)
K_all = self.k_proj(X_conv)      # Shape: (B, L, H * d_k)
V_all = self.v_proj(X_conv)      # Shape: (B, L, H * d_v)
Gates_all = self.W_gates(X_conv)  # Shape: (B, L, 3 * H)

% Sequential recurrence loop slicing pre-computed inputs
for t in range(L):
    q_t = Q_all[:, t, :]
    k_t = K_all[:, t, :]
    v_t = V_all[:, t, :]
    gates_t = Gates_all[:, t, :]
    % Perform element-wise state cell updates
\end{lstlisting}
This reduces recurrent computation to element-wise updates, bypassing GPU latency bottlenecks.

\section{Conclusion}
Our research demonstrates that a Causal Transformer equipped with Sinusoidal Positional Encodings is capable of performing "Surgical Calculus" on its own history, mathematically deriving required velocities for balance under partially observable conditions. Causal Transformers consistently outperform reactive MLP baselines in Partially Observable Markov Decision Processes (POMDP) and RLHF training. The implementation of structural modifications---such as gradient clipping, Polyak target updates, normalizer corrections, scheduled action masking, and Jury consensus models---stabilized training and resolved performance collapse. These results highlight the importance of temporal context, structured attention, and numerical stabilization in designing robust, sequence-conditioned agents for continuous control.
